bibliotek.dk custom search

The module customize the default search by adding form elements to the search-block-form, based on the page path.

In the administration interface you can edit the values of form elements, make nested checkbox or radio button values, combine form elements into a single search element, add search elements to a search page, and assign a search page to a path.

Search pre-process and javascript for combined form values are mantained seperately, in bibdk_custom_search_preprocess (or any module that implements hook_bibdk_custom_search_field_preprocess().)

Form values

When adding custom values for searchpages, use english cql language (even though the rest might be written in another language eg. danish) e.g. (fra or fre)

Search code
The search code will be combined with the values to form a search query, f.ex.:
Search code: term.type + Value: cd-rom => term.type=cd-rom
Search code: term.type + Value: (avis or tidsskrift) => term.type=(avis or tidsskrift)

If the value list is a submenu, the Search code will be inherited from the parent value list.

If the search code is not used in EVERY value, leave it empty, and type in the query string in the value. f.ex.:
Search code: [blank] + Value: term.type.bog or term.workType=literature

Facet lists
You can create facet lists by defining an option value as facet.[type] e.g. facet.language.
If searchcode is empty write [searchcode]=facet.[type] e.g. term.type=facet.type
